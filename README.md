A passive vulnerability assessment was conducted on the target web application using automated scanning tools (Nmap, OWASP ZAP) and manual browser-based inspection. The objective was to identify common security misconfigurations and information disclosure issues without performing any intrusive or exploitative testing.

The assessment identified several security weaknesses related to missing HTTP security headers, session cookie configuration, and information disclosure through server responses. While no critical exploitable vulnerabilities were confirmed, the findings indicate moderate weaknesses in security hardening and baseline web protection controls.

Overall, the system demonstrates a moderate security posture, with improvements required in browser security enforcement, transport security configuration, and attack surface reduction.
